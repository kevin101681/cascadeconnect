## üìù Maintenance

### Adding New Documentation
- Place new documentation in `docs/active/`
- Use clear, descriptive filenames
- Include the date or version in comprehensive guides

### Archiving Documentation
Move documentation to `docs/archive/` when:
- A newer, more comprehensive version exists
- The feature has been completely replaced
- The documentation is no longer relevant to current implementation
- The document represents a completed, historical task

### Updating This README
When the active documentation changes significantly:
1. Update the file counts
2. Add/remove entries from the active list
3. Update category descriptions if needed
